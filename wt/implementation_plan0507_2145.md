# G1:Mちゃんのアクション類推と左手モーション修正プラン

ユーザーが「左手上げて」と言っても反映されない問題の解決と、首をかしげたり踊ったりといった類推動作を可能にするための修正を行います。

## ユーザーレビューが必要な事項
- モーションの類推は、LLMの回答テキストやタグから判断されます。LLMが不適切なアクション名を出力した場合、デフォルトの動作（少し体を揺らすなど）になります。

## 修正内容

### 1. アクション検出の柔軟化 (frontend/src/MainApp.tsx)
- `knownActions` による厳格なチェックを廃止し、スネークケースであればどのアクション名も受け入れるようにします。
- LLMの回答から「左」「右」といったキーワードを検出し、`left_raise` や `right_raise` として解釈するロジックを強化します。
- タグがある場合でも、テキスト内に「左手」などの指示があればそれを優先・統合します。

### 2. executeBotAction の拡張 (frontend/src/MainApp.tsx)
- `tilt_head`（首をかしげる）: 頭をZ軸で揺らします。
- `dance`: 既存の微細な揺れから、よりダイナミックな左右への揺れやジャンプを組み合わせた動作に変更します。
- `thinking`: 頭をX軸で傾け、少し左右に振る動作を追加します。

### 3. モーションの鏡像修正 (frontend/src/MainApp.tsx)
- 左腕と右腕の回転方向（Z軸）が鏡像になっている場合を考慮し、左右で適切な回転量を計算するように修正します。
- 現在、左右共に `-1.2` から `1.3` へ変化させていますが、右腕が逆方向に動いている可能性があるため、これを修正します。

### 4. フォールバック正規表現の強化 (frontend/src/MainApp.tsx)
- 日本語のテキストからアクションを類推するための正規表現に「首をかしげる」「考える」「わくわく」などを追加します。

## 検証プラン

### 自動テスト/手動検証
- LLMへの入力として「左手を上げて」「首をかしげて」「ダンスして」を送り、アバターが期待通りの動作をするか目視で確認します。
- ブラウザのコンソールログに出力される `Executing bot action: ...` を確認し、意図したアクション名が渡されているかチェックします。
